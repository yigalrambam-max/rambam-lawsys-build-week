# Pre-existing Work and Build Week Scope

## Pre-existing project

Before OpenAI Build Week, Rambam LawSys existed as an evolving legal-learning methodology and controlled Custom GPT workflow.

The pre-existing work included source fidelity, source-specific legal analysis, separation between source and interpretation, controlled synthesis, quality governance, and structured legal-learning products.

## Work performed during the submission period

During July 13-21, 2026, the project was extended and packaged for public evaluation through work that included:

- A public, non-confidential demonstration repository
- A source-coverage and readiness demo
- Public sample data and testing instructions
- A workflow diagram
- A Devpost project description and judge-testing pathway
- A short demo video and human narration
- Public documentation of GPT-5.6 collaboration
- Codex-based repository review and implementation work, to be documented from the actual primary Codex session
- Repository commits and a `/feedback` Session ID from that primary Codex session

## Evaluation boundary

The submission does not claim that the full Rambam LawSys methodology was created during Build Week. Evaluators should assess the concrete public demonstration, documentation, tests, repository improvements, and product extensions completed during the official submission period.

## Confidentiality boundary

The public repository excludes private Custom GPT instructions, confidential knowledge files, copyrighted course readings, personal student information, API keys, credentials, and internal governance artifacts not approved for publication.
