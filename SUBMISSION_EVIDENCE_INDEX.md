# Submission Evidence Index

| Evidence | Location | Purpose |
|---|---|---|
| Project overview and test instructions | `README.md` | Explains the public demo and judge pathway |
| Public workflow diagram | `assets/` or repository root | Explains inputs, controls, and outputs |
| Public source-coverage demo | `index.html`, `app.js`, `styles.css` | Demonstrates source intake and readiness decisions |
| Sample data | `sample-data/` | Provides a reproducible judge test |
| Manual test instructions | `manual-test.html` or `tests/` | Explains expected GO and NO-GO behavior |
| Build Week implementation log | `BUILD_WEEK_LOG.md` or `docs/BUILD_WEEK_LOG.md` | Records verified submission-period changes |
| GPT-5.6 evidence | `docs/GPT5_6_COLLABORATION_EVIDENCE.md` | Records substantive GPT-5.6 collaboration |
| Scope disclosure | `docs/PREEXISTING_VS_BUILD_WEEK_SCOPE.md` | Separates pre-existing work from Build Week work |
| Codex session evidence | Primary Codex session and `/feedback` ID | Records actual Codex development work |
| Git history | Repository commits and pull requests | Provides dated implementation evidence |
| Demo video | Public YouTube link in Devpost | Shows the product and explains the workflow |

## Private evidence retained by project owner

- Timestamped ChatGPT project conversations
- Internal development and QA records
- Original generated artifacts
- Video narration and parity records
- Non-public methodology and governance materials

## Evidence limitations

ChatGPT conversation history supports the GPT-5.6 collaboration claim but does not replace Codex implementation evidence, the required `/feedback` Session ID, repository commits, test results, or a working public demo.
